# 정부과제 자료조사 카테고리 분류 체계

> 기업 입장에서 정부과제 자료조사는 **공고를 많이 모으는 것**보다 **우리 회사가 실제로 지원할 수 있고, 사업적으로 활용 가능한 과제를 빠르게 선별하는 것**이 핵심이다.  
> 따라서 단순히 부처별로 나누기보다는 **사업 목적 → 기업 적합성 → 과제 규모 → 수행 방식 → 기술 분야 → 실무 리스크 → 우선순위** 순서로 분류하는 것이 가장 효율적이다.

---

## 1. 전체 분류 구조 요약

```text
A. 사업 목적
   ├─ R&D/기술개발
   ├─ 실증/PoC
   ├─ 사업화/판로
   ├─ 창업/스케일업
   ├─ 인력/고용
   ├─ 수출/해외진출
   └─ 바우처/컨설팅

B. 기업 적합성
   ├─ 스타트업
   ├─ 중소기업
   ├─ 중견기업
   ├─ 대기업
   ├─ 대학/연구소 협력 필요
   └─ 지역 제한 여부

C. 과제 규모
   ├─ 소형: ~5천만 원
   ├─ 중소형: 5천만~2억 원
   ├─ 중형: 2억~10억 원
   ├─ 대형: 10억 원 이상
   └─ 초대형: 수십억 원 이상

D. 수행 방식
   ├─ 단독형
   ├─ 산학연 협력형
   ├─ 컨소시엄형
   ├─ 수요연계형
   ├─ 실증형
   ├─ 바우처형
   ├─ 지정공모형
   └─ 자유공모형

E. 기술/산업 분야
   ├─ AI/Agent
   ├─ 데이터
   ├─ SaaS/클라우드
   ├─ 보안
   ├─ 제조
   ├─ 헬스케어
   ├─ 금융
   ├─ 공공/국방
   └─ 기타 산업 도메인

F. 실무 리스크
   ├─ 민간부담금
   ├─ 기술료
   ├─ 정산 부담
   ├─ 실증처 필요
   ├─ 컨소시엄 필요
   ├─ IP/성과물 귀속
   ├─ 보안/개인정보
   └─ 마감 임박 여부
```

# 2. 1차 분류: 사업 목적 기준

가장 먼저 나눠야 하는 기준은 **사업 목적**이다.  
기업은 “무슨 돈을 받을 수 있는가?”보다 **이 과제가 우리 회사의 어떤 문제를 해결해주는가**를 기준으로 봐야 한다.

| 대분류 | 목적 | 기업 입장에서 의미 |
|---|---|---|
| R&D/기술개발 | 신기술, 신제품, 알고리즘, 플랫폼 개발 | 핵심 기술 고도화 |
| 실증/PoC | 실제 현장 적용, 테스트베드 검증 | 레퍼런스 확보 |
| 사업화 | 제품화, 판로, 마케팅, 인증, 양산 | 매출화 지원 |
| 창업/성장지원 | 초기창업, 도약, 스케일업 | 스타트업 성장자금 |
| 수출/해외진출 | 전시회, 현지화, 해외 인증 | 글로벌 진출 |
| 인력/고용 | 채용, 교육, 연구인력, 청년고용 | 인건비 부담 완화 |
| 인프라/바우처 | 클라우드, AI, 데이터, 컨설팅 바우처 | 비용 절감 |
| 정책/지역특화 | 지자체, 규제특구, 지역산업 | 지역 기반 기업에 유리 |

## 실무 판단 기준

- 기술을 고도화하고 싶다면: **R&D/기술개발**
- 고객사 레퍼런스가 필요하다면: **실증/PoC**
- 제품 판매·마케팅이 목적이라면: **사업화/판로**
- 초기 자금 확보가 필요하다면: **창업/성장지원**
- 비용 부담을 줄이고 싶다면: **바우처/인프라 지원**

# 3. 2차 분류: 기업 성장단계 기준

같은 AI 과제라도 예비창업자, 초기 스타트업, 중소기업, 중견기업, 대기업이 접근할 수 있는 과제는 다르다.  
따라서 자료조사 단계에서 **기업 성장단계별로 신청 가능성을 먼저 필터링**해야 한다.

| 성장단계 | 봐야 할 과제 |
|---|---|
| 예비창업 | 예비창업패키지, 창업교육, 시제품 제작 |
| 초기창업 1~3년 | 초기창업패키지, 창업성장기술개발, 바우처 |
| 도약기 3~7년 | 창업도약패키지, TIPS 연계, 스케일업 R&D |
| 중소기업 | 중기부 R&D, 제품화, 공정혁신, 인력지원 |
| 중견기업 | 산업부 R&D, 실증, 공급망, 컨소시엄형 과제 |
| 대기업 | 국가전략기술, 대형 실증, 수요기업, 컨소시엄 |
| 대학/연구소 협력기업 | 산학연 R&D, 공동연구, 기술이전 |

## 기업 입장에서 중요한 질문

- 우리 회사가 주관기관으로 신청 가능한가?
- 공동기관 또는 수요기업으로만 참여 가능한가?
- 업력 제한이 있는가?
- 매출 규모 제한이 있는가?
- 중소기업 전용 과제인가?
- 대기업 참여가 가능한 과제인가?

# 4. 3차 분류: 과제 규모 기준

과제 규모는 지원금뿐 아니라 **작성 난이도, 행정 부담, 컨소시엄 필요성, 성공확률**에 직접 영향을 준다.

| 규모 | 정부지원금 기준 예시 | 특징 | 적합한 기업 |
|---|---:|---|---|
| 소형 | 수백만~5천만 원 | 바우처, 컨설팅, 인증, 전시회 | 초기기업, 리소스 부족 기업 |
| 중소형 | 5천만~2억 원 | 시제품, PoC, 초기 기술개발 | 스타트업, 중소기업 |
| 중형 | 2억~10억 원 | 본격 R&D, 제품화, 실증 | 기술기업, 연구인력 보유 기업 |
| 대형 | 10억 원 이상 | 컨소시엄, 다년도, 국가전략기술 | 중견·대기업, 대학·출연연 협력 |
| 초대형 | 수십억~수백억 원 | 산업 생태계, 인프라, 플랫폼, 실증단지 | 대기업/연구기관 중심 |

## 규모 판단 시 주의점

금액이 클수록 무조건 좋은 것은 아니다.

- 작은 기업이 대형 과제를 주관하면 행정·정산·협약·보고 부담이 과도할 수 있다.
- 연구조직이 있는 기업은 중형 이상 과제에서 실익이 크다.
- 대기업이나 중견기업은 소형 과제보다 실증형, 컨소시엄형, 전략기술형 과제에서 효과가 크다.
- 과제 규모가 클수록 컨소시엄, 실증처, 정량성과, 사업화계획 요구 수준이 높아진다.

# 5. 4차 분류: 수행 방식 기준

수행 방식은 자료조사에서 가장 실무적인 분류다.  
같은 R&D 과제라도 단독 수행인지, 컨소시엄 수행인지에 따라 준비물이 완전히 달라진다.

| 수행 방식 | 설명 | 체크 포인트 |
|---|---|---|
| 단독형 | 기업 혼자 신청/수행 | 내부 인력, 기술역량, 사업화 계획 |
| 산학연 협력형 | 기업+대학+연구소 | 역할분담, IP, 논문/특허, 연구책임자 |
| 수요연계형 | 수요기업이 실증/구매 검토 | 수요처 확보, LOI/MOU |
| 실증형 | 현장 적용 중심 | 테스트베드, 데이터, 운영환경 |
| 컨소시엄형 | 여러 기관 공동 수행 | 주관기관 역량, 예산 배분, 총괄관리 |
| 위탁형 | 일부 업무 외부 위탁 | 위탁 필요성, 결과물 명확성 |
| 바우처형 | 공급기업 서비스 이용 | 빠른 신청, 비용절감, 정산 간소화 |
| 지정공모형 | RFP가 정해진 과제 | RFP 적합성, 기술해석 능력 |
| 자유공모형 | 주제를 직접 제안 | 기획력, 차별성, 시장성 |

## 수행 방식별 준비 난이도

| 수행 방식 | 준비 난이도 | 특징 |
|---|---:|---|
| 바우처형 | 낮음 | 빠르게 신청 가능, 비용절감 목적 |
| 단독형 | 중간 | 내부 역량이 중요 |
| 자유공모형 | 중간~높음 | 과제기획 역량이 중요 |
| 실증형 | 높음 | 실증처와 데이터 확보 필요 |
| 산학연 협력형 | 높음 | 기관 간 역할·IP 조율 필요 |
| 컨소시엄형 | 매우 높음 | 총괄관리, 예산 배분, 행정 부담 큼 |
| 지정공모형 | 매우 높음 | RFP 요구사항 충족이 핵심 |

# 6. 5차 분류: 기술/산업 분야 기준

기술 분야는 회사의 제품전략과 직접 연결해야 한다.  
단순히 “AI 과제”로 묶으면 너무 넓기 때문에, 내부적으로는 **제품 모듈 또는 기술 스택 기준**으로 나누는 것이 좋다.

## 일반 기술 분야 분류

| 기술 카테고리 | 세부 키워드 |
|---|---|
| AI/생성형 AI | LLM, Agent, RAG, 멀티에이전트, 평가, AI 안전성 |
| 데이터 | 데이터 구축, 품질관리, 데이터댐, 마이데이터, 합성데이터 |
| 클라우드/SaaS | SaaS 전환, 클라우드 네이티브, API 플랫폼 |
| 보안 | 개인정보보호, 제로트러스트, AI 보안, 산업보안 |
| 제조 AI | 스마트팩토리, 공정 최적화, 예지보전, 품질검사 |
| 헬스케어 AI | 의료영상, 디지털헬스, 임상검증, 의료기기 |
| 금융 AI | 이상거래탐지, 리스크관리, 문서자동화, 규제 대응 |
| 로봇/자율주행 | 인지, 제어, 경로계획, 시뮬레이션 |
| 콘텐츠/미디어 | 생성형 콘텐츠, 저작권, 버추얼휴먼 |
| 공공/안전 | 재난, 치안, 국방, 사회문제 해결 |

## AI/Agent 기업 기준 세부 분류

AI Agent 기업이라면 아래처럼 더 세밀하게 분류하는 것이 좋다.

```text
1. Agent Orchestration
2. Tool/API Integration
3. RAG/Knowledge Base
4. Agent Evaluation/Monitoring
5. Security/Governance
6. Workflow Automation
7. Vertical Use Case
   ├─ 금융
   ├─ 제조
   ├─ 공공
   ├─ 법무
   ├─ 고객센터
   └─ 연구/문서 자동화
```

## AI/Agent 기업 기준 공고 매칭 예시

| 공고 유형 | 우선순위 | 이유 |
|---|---|---|
| 생성형 AI 기반 업무자동화 R&D | A | 핵심 기술과 직접 연결 |
| SaaS 전환 지원사업 | A | 제품화/상용화에 도움 |
| AI 바우처 공급기업/수요기업 | A/B | 매출 또는 고객 확보 가능 |
| 제조 AI 실증사업 | B | 제조 수요기업 필요 |
| 데이터 구축사업 | B | RAG/학습데이터와 연결 가능 |
| 창업패키지 | 기업 업력 따라 A/B | 자금·멘토링·사업화 지원 |
| 해외 전시회 지원 | B/C | 제품 성숙도에 따라 다름 |
| 일반 경영 컨설팅 | C | 기술기업 핵심성과와 약함 |
| 지역 소상공인 지원 | 제외 | 타깃 불일치 가능성 높음 |

# 7. 6차 분류: 부처/전문기관 기준

부처 기준은 1차 분류로 쓰기보다는 **공고 탐색과 접수 채널 관리용**으로 쓰는 것이 좋다.

| 부처/기관 | 주로 보는 과제 |
|---|---|
| 과기정통부 / IITP / NRF | AI, SW, ICT, 기초·원천연구 |
| 산업부 / KEIT / KIAT | 산업기술, 제조, 소재, 부품, 장비, 실증 |
| 중기부 / TIPA | 중소기업 R&D, 창업성장, 공정혁신 |
| NIPA | SW, SaaS, AI 바우처, 디지털 전환 |
| 창업진흥원 | 창업패키지, 창업사업화 |
| 보건복지부 / KHIDI | 바이오, 헬스케어, 의료기기 |
| 국토부 / KAIA | 모빌리티, 스마트시티, 건설기술 |
| 환경부 / KEITI | 탄소중립, 환경기술 |
| 지자체/TP | 지역산업, 실증, 기업지원 |
| 방사청/국방기술기관 | 국방, 보안, 미래전장 |

## 부처 기준 분류의 한계

부처별로만 나누면 다음 문제가 생긴다.

- 실제 지원 가능 여부를 빠르게 판단하기 어렵다.
- 같은 AI 과제라도 R&D, 실증, 바우처, 사업화 성격이 섞여 있다.
- 기업 성장단계와 맞지 않는 과제를 계속 보게 된다.
- 컨소시엄 필요 여부, 민간부담금, 정산 부담 같은 실무 리스크가 가려진다.

따라서 부처 기준은 **보조 분류**로만 활용하고, 1차 기준은 사업 목적과 기업 적합성으로 잡는 것이 좋다.

# 8. 7차 분류: 신청 자격/제외 조건 기준

자료조사 엑셀이나 노션에서 반드시 필드로 만들어야 하는 영역이다.  
이 기준을 빠르게 확인하지 않으면, 지원할 수 없는 과제에 시간을 낭비하게 된다.

| 필드 | 확인 이유 |
|---|---|
| 기업 규모 | 중소/중견/대기업 가능 여부 |
| 업력 | 창업 3년/7년 제한 여부 |
| 매출 조건 | 매출 상한/하한 |
| 지역 제한 | 특정 지역 소재 기업만 가능 여부 |
| 주관 가능 여부 | 우리 회사가 주관 가능한지 |
| 공동/참여 가능 여부 | 컨소시엄 참여만 가능한지 |
| 민간부담금 | 현금 부담 가능 여부 |

| 중복지원 제한 | 기존 과제와 충돌 여부 |
| 참여제한/세금체납 | 행정 리스크 |
| TRL 조건 | 현재 기술성숙도와 맞는지 |
| 실증처 필수 여부 | 고객사/수요처 필요 여부 |

## 빠른 제외 기준

- 신청자격 미충족
- 지역 제한 불일치
- 업력/매출 조건 불일치
- 민간부담금 과도
- 기존 과제와 중복 위험
- 실증처 필수인데 확보 불가
- 컨소시엄 필수인데 파트너 없음
- 마감 임박
- 내부 인력으로 수행 불가

|---|---:|

| 신청 자격 충족 가능성 | 0~5점 |

| 행정/정산 부담 | 0~5점 |
| 사업화/레퍼런스 효과 | 0~5점 |
| 컨소시엄 확보 난이도 | 0~5점 |
| 마감까지 남은 기간 | 0~5점 |

## 점수 해석 방식

평가 항목은 두 가지로 나눠서 봐야 한다.

### 긍정 점수

- 기술/제품 적합도
- 신청 자격 충족 가능성
- 정부지원금 규모
- 선정 가능성
- 사업화/레퍼런스 효과
- 마감까지 남은 기간

### 리스크 점수

- 작성 난이도
- 행정/정산 부담
- 컨소시엄 확보 난이도
- 민간부담금 부담

즉, 전체 점수가 높아도 리스크 점수가 너무 높으면 실제 지원 우선순위는 낮아질 수 있다.

# 10. 우선순위 분류 기준

정부과제 자료조사 결과는 최종적으로 **A/B/C/제외**로 나누는 것이 좋다.

## A급: 바로 검토해야 하는 과제

조건:

- 우리 제품/기술과 직접 연결됨
- 신청 자격이 명확히 충족됨
- 지원금 규모가 의미 있음
- 민간부담금 감당 가능
- 마감까지 2주 이상 남음
- 단독 또는 이미 아는 파트너로 가능
- 사업화/레퍼런스 효과가 큼

예시:

- 생성형 AI 기반 업무자동화 R&D
- SaaS 전환 지원사업
- AI 바우처 공급기업/수요기업 참여
- 기존 고객사와 바로 실증 가능한 PoC 과제

---

## B급: 파트너 있으면 검토할 과제

조건:

- 기술 분야는 맞음
- 컨소시엄 또는 실증처가 필요함
- 수요기업/대학/연구소가 필요함
- 마감까지 시간이 애매함
- 작성 난이도가 높음
- 선정 가능성은 중간 수준

예시:

- 제조 AI 실증사업
- 데이터 구축사업
- 산학연 공동 R&D
- 지정공모형 대형 과제

---

## C급: 모니터링만 할 과제

조건:

- 기술 분야는 인접하지만 직접적이지 않음
- 지원 규모가 작음
- 행정 부담이 큼
- 사업화 효과가 낮음
- 자격조건이 불확실함
- 당장 리소스를 투입하기 애매함

예시:

- 일반 경영 컨설팅
- 단기 교육지원
- 제품과 직접 연결되지 않는 전시회 지원
- 장기적으로만 의미 있는 정책사업

---

## 제외

조건:

- 신청자격 미충족
- 지역 제한 불일치
- 업력/매출 조건 불일치

# 12. 공고 검토용 체크리스트

## 12.1 사업 목적 체크

- 이 과제가 R&D인가, 실증인가, 사업화인가?
- 우리 회사가 지금 필요한 목적과 맞는가?
- 과제 종료 후 실제 제품/서비스와 연결되는가?
- 단순 지원금 확보보다 전략적 효과가 있는가?

## 12.2 신청 자격 체크

- 우리 회사 규모가 지원 대상에 포함되는가?
- 업력 제한을 충족하는가?
- 매출, 지역, 업종 제한이 없는가?
- 주관기관으로 신청 가능한가?
- 공동기관 또는 수요기업으로만 가능한가?

## 12.3 기술 적합도 체크

- 우리 핵심 기술과 직접 연결되는가?
- 기존 제품을 고도화하는 데 도움이 되는가?
- 내부 인력으로 수행 가능한 범위인가?
- 추가 파트너가 필요한가?
- 정량 목표를 달성할 근거가 있는가?

## 12.4 사업화/레퍼런스 체크

- 고객사 확보에 도움이 되는가?
- PoC 또는 실증 레퍼런스로 활용 가능한가?
- 매출 모델과 연결되는가?
- 과제 종료 후 상용화 계획이 명확한가?

## 12.5 리스크 체크

- 민간부담금이 과도하지 않은가?
- 정산·보고 부담을 감당할 수 있는가?
- 기술료 조건이 부담되지 않는가?
- IP/성과물 귀속 조건이 불리하지 않은가?
- 개인정보/보안 리스크가 있는가?
- 마감까지 작성 시간이 충분한가?

# 14. 기업 입장에서 가장 효율적인 분류 결론

정부과제 자료조사는 단순히 “부처별/공고별”로 나누면 비효율적이다.  
기업 입장에서는 아래 순서로 분류하는 것이 가장 효율적이다.

```text
1. 사업 목적
   → 이 과제가 우리 회사의 어떤 목적을 해결하는가?

2. 신청 자격
   → 우리 회사가 실제로 지원 가능한가?

3. 과제 규모
   → 지원금과 행정 부담이 적절한가?

4. 수행 방식
   → 단독인가, 컨소시엄인가, 실증처가 필요한가?

5. 기술 적합도
   → 우리 제품/기술과 직접 연결되는가?

6. 실무 리스크
   → 민간부담금, 정산, 기술료, IP, 마감 리스크는 어떤가?

7. 우선순위
   → A/B/C/제외로 나눠서 실행 여부를 판단한다.
```

# 16. 한 줄 결론

기업 입장에서 정부과제 자료조사의 핵심은 **공고를 많이 찾는 것**이 아니라, **우리 회사의 기술·제품·사업전략과 맞는 과제를 빠르게 선별하고 A/B/C/제외로 실행 우선순위를 정하는 것**이다.
