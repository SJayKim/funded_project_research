# Documentation Map

- `audits/`: coverage and implementation audits.
- `planning/`: active TODO and planning notes.
- `research/`: source, category, and UI/UX research.
- `sessions/`: dated working-session context.
- `specs/`: collection and alert-agent specifications.

Runtime code lives in `funded_project_research/`.
Run modules from the repository root with `python -m funded_project_research.<module>`.
